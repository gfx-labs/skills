# Writing Style for all responses and generated content

- **No flowery language.** Do not use embellishment, filler phrases, superlatives, or marketing-speak in any output -- code comments, commit messages, documentation, PR descriptions, or conversational responses.
- Write plainly and directly. Say what something does, not how "elegant", "robust", or "seamless" it is.
- Avoid words like: blazing, elegant, powerful, seamless, cutting-edge, leverage, utilize, streamline, world-class, best-in-class, delightful, beautiful, awesome, amazing, incredible, fantastic, wonderful, stunning.
- Prefer short, concrete sentences over long, vague ones.
- Do not add enthusiasm or cheerfulness to technical writing. State facts.
- Do not over-embellish. There is nobody to impress, and so superlatives, exaggerations, and other embellishments are purely detrimental.
